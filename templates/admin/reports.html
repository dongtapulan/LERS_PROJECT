How the Operating System Supports LERS

1. Involved OS Functions

  • File Management: The OS handles the storage and retrieval of equipment images and PDF reservation slips in the local file system.
  • Process Management: The OS allocates CPU time and memory to the Python interpreter and PostgreSQL server to handle simultaneous requests.
  • Network Management: The OS TCP/IP stack enables communication between the Flask server and client browsers.

2. Potential OS Problems

  • Concurrency Issues: Race conditions when two users try to reserve the last available item at the same millisecond.
  • Memory Leaks: Improper process handling that could eventually exhaust server RAM.
  • Storage Overload: Unmanaged generation of PDF slips filling up disk space.

3. Implementation & Handling

To handle these, LERS utilizes Database Locking provided by the OS-backed PostgreSQL engine to prevent race conditions. We implement Secure Filename Handling via the Werkzeug library to prevent OS-level directory traversal attacks, and utilize the OS's Multithreading capabilities to handle multiple user sessions without blocking the main system process.
